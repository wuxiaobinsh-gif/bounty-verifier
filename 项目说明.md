# Bounty Claim Verifier (赏金验证机器人)

## 项目简介

Bounty Claim Verifier 是一个 GitHub Action 机器人，自动验证赏金任务的提交评论。它可以检查用户的 star 数量、follow 状态、钱包余额、文章链接有效性，以及检测重复领取，从而消除人工验证的时间成本。

## 功能特性

1. **Star/Follow 验证** — 通过 GitHub API 检查用户是否 star 了指定仓库并 follow 了仓库所有者
2. **钱包余额检查** — 查询 RustChain 节点 API 验证钱包是否存在及余额
3. **文章链接验证** — 验证 dev.to/Medium 文章链接是否可访问
4. **文章质量评估** — 统计文章字数、检查代码示例和结构化程度
5. **重复领取检测** — 扫描历史评论，检测同一用户是否已领取过赏金
6. **自动报告生成** — 生成格式化的验证报告并作为评论发布

## 技术栈

- Python 3.11
- GitHub Actions
- GitHub REST API
- RustChain Node API
- YAML 配置

## 使用方式

### 1. 安装到你的仓库

将 `.github/workflows/verify-claim.yml` 复制到你的仓库，确保 `src/` 目录也一并复制。

### 2. 配置 Secrets

在仓库设置中添加：
- `GITHUB_TOKEN` — GitHub 自动提供
- `RUSTCHAIN_NODE_URL` — 可选，默认 `https://50.28.86.131`

### 3. 自定义规则

编辑 `config/bounty-rules.yml` 来修改：
- 每个 star 的 RTC 奖励
- follow 奖励
- 最低 star 数要求
- 重复领取检测窗口

### 4. 手动测试

```bash
cd bounty-verifier
pip install -r requirements.txt
GITHUB_TOKEN=xxx python src/main.py
```
